# LeakGuard File Capability Matrix

This matrix is the authoritative user-facing scope for current file protection. All processing listed here is local-only in the browser or extension package. LeakGuard does not use remote OCR, backend file processing, telemetry, cloud verification, or remote model calls.

| File/input type | Scanner support | Protected-site support | Output type | Fail-closed behavior | Unsupported limitations | Raw-data safety notes |
| --- | --- | --- | --- | --- | --- | --- |
| UTF-8 text, source, config, logs, and data files (`.txt`, `.md`, `.env`, `.log`, `.json`, `.yaml`, `.yml`, `.toml`, `.xml`, `.csv`, `.ini`, `.conf`, `.cfg`, `.pem`, `.key`, shell/source files, `Dockerfile`, `Makefile`) | Yes. Scanned locally with deterministic redaction. | Yes, where paste/drop/file-select handoff can be safely intercepted and replaced. Large files use streaming/chunked redaction up to the local limit. | Scanner: redacted text copy and sanitized JSON report. Protected sites: sanitized text file with the original supported text filename where possible. | Invalid UTF-8, binary-looking content, over-limit files, and failed sanitized handoff are blocked rather than uploaded raw after LeakGuard attempts sanitization. | Single-file scanner flow; not every site/browser upload control accepts synthetic sanitized files. | Raw contents stay in memory/chunks during processing. Raw file text is not intentionally stored in extension storage, localStorage, sessionStorage, cache debug snapshots, audit metadata, logs, or JSON reports. |
| Text PDF (`.pdf`) | Yes, text extraction only. Scanner can export `.redacted.txt` plus a regenerated `.redacted.pdf` from sanitized extracted text. | Yes, text extraction only. | Scanner: `.redacted.txt` and regenerated `.redacted.pdf`. Protected sites: regenerated `.redacted.pdf` only when complete, otherwise `.redacted.txt` fallback. | Malformed, encrypted, scanned/image-only, no-text, and over-limit PDFs fail closed with no raw upload through protected-site paths and no `.redacted.pdf` output. Truncated regenerated PDFs are not handed off. | No scanned-PDF OCR, no visual PDF redaction, no PDF structure or layout preservation. Protected-site PDF output is regenerated from sanitized text only and is not layout-preserving. | Extracted text is redacted before export/handoff. Regenerated PDFs are built from sanitized text only; original PDF bytes/streams are not copied. Metadata and reports must not include raw detected secrets. |
| DOCX (`.docx`) | Yes, text extraction only. Scanner can export `.redacted.txt` plus a regenerated `.redacted.docx` from sanitized extracted text. | Yes, text extraction only. | Scanner: `.redacted.txt` and regenerated `.redacted.docx`. Protected sites: regenerated `.redacted.docx` only when complete, otherwise `.redacted.txt` fallback. | Malformed, encrypted, no-text/image-only, unsupported compression, and over-limit DOCX files fail closed with no raw upload and no `.redacted.docx` output. Truncated regenerated DOCX files are not handed off. | No legacy `.doc`, no `.docm`, no macros, no embedded media OCR/redaction, no tracked-change reconstruction, and no DOCX format or layout preservation. | Extracted text is redacted before export/handoff. Regenerated DOCX files are built from sanitized text only; original DOCX XML parts are not copied, and original styles, images, comments, and metadata are not preserved. Raw document text is not cached or persisted. |
| XLSX (`.xlsx`) | Yes, workbook/shared-string/worksheet text extraction. Formulas are scanned as text only and are not executed. Scanner can export `.redacted.txt` plus a simple regenerated `.redacted.xlsx` from sanitized extracted text. | Yes, text extraction only. Protected-site `.redacted.xlsx` handoff is supported only when regenerated output is complete. | Scanner: `.redacted.txt` and regenerated `.redacted.xlsx`. Protected sites: regenerated `.redacted.xlsx` only when complete, otherwise `.redacted.txt` fallback. | Malformed, encrypted, no-text/image-only, unsupported compression, legacy/macro formats, and over-limit XLSX files fail closed with no raw upload and no `.redacted.xlsx` output. Truncated regenerated XLSX files are not handed off. | No `.xls`, `.xlsm`, `.xlsb`, `.xltm`, macro execution, chart/media OCR, formula preservation, comments, hidden-sheet preservation, metadata/custom XML/media preservation, calc-chain preservation, cell-format preservation, or layout preservation. | Extracted text is redacted before export/handoff. Formula text is treated as text. Regenerated scanner and protected-site XLSX output is generated from sanitized extracted text only and does not copy original OOXML spreadsheet parts. Raw spreadsheet data is not cached or persisted. |
| Image metadata (`.png`, `.jpg`, `.jpeg`, `.webp`) | Yes. Metadata scan includes safe file metadata such as filename, extension, MIME type, and size bucket. | Yes as metadata-only when protected-site OCR is turned off. | `.redacted.txt` for metadata findings. | Unsupported image types or unsafe paths fail closed where LeakGuard cannot safely continue. | Metadata scan is not visual OCR. Pixel content is scanned only in the scanner OCR flow or the protected-site OCR path. | Pixel bytes are not included in metadata reports. Raw secret filenames are redacted from outputs and metadata. |
| Scanner image OCR (`.png`, `.jpg`, `.jpeg`, `.webp`) | Yes, English-only OCR after the user selects an image and starts scanning. | Protected-site OCR is settings-controlled and enabled by default for supported image uploads. | Redacted text export; eligible visual redaction exports a flattened `.redacted.png`. | OCR timeout, corrupt images, oversized images, unsupported types, missing boxes, low-confidence boxes, or redaction failures fail closed for visual export. | No non-English OCR, no scanned-PDF OCR, no remote OCR, no original image format preservation for visual redaction. JPG/JPEG/WEBP visual redaction exports PNG. | OCR text and redaction boxes are sanitized before reports/debug metadata. Raw OCR/image bytes are not intentionally stored in extension storage, localStorage, sessionStorage, logs, audit metadata, or cache snapshots. |
| Protected-site image OCR (`.png`, `.jpg`, `.jpeg`, `.webp`) | N/A | Settings-controlled and enabled by default. OCR is local, English-only, images-only, and uses visual `.redacted.png` upload only when OCR boxes are eligible. Users can turn it off in settings. | Eligible boxes: `.redacted.png`. Fallback/non-eligible boxes: `.redacted.txt` when text-only redaction is safe. | OCR runtime failure, timeout, unsafe/low-confidence boxes, or visual redaction failure blocks raw image upload. | No scanned-PDF OCR, no non-English OCR, no remote OCR/backend, no image format preservation. | Raw image bytes and OCR text must not leave the device or appear in logs, reports, audit metadata, storage, cache debug snapshots, or page DOM outside intended sanitized result surfaces. |
| Archives, executables, legacy/macro Office files, unsupported images, and binary files | No. | No supported redaction path. | None. | Unsupported paths are not represented as scanned, sanitized, or protected; protected paths block when LeakGuard cannot safely pass through. | No ZIP/7z/RAR/TAR/GZ, EXE/DLL/MSI/DMG/PKG/APP, legacy `.doc`/`.xls`, macro formats, GIF/BMP/ICO/SVG, or arbitrary binary redaction. | Unsupported file contents are not scanned or exported by LeakGuard. |

Release audit gates for this matrix:

- Scanner and protected-site text PDFs can export regenerated `.redacted.pdf` from sanitized text only; protected-site PDFs fall back to `.redacted.txt` when regeneration would truncate. Scanner and protected-site DOCX can export regenerated `.redacted.docx` from sanitized text only; protected-site DOCX falls back to `.redacted.txt` when regeneration would truncate. Scanner and protected-site XLSX can export regenerated `.redacted.xlsx` from sanitized text only; protected-site XLSX falls back to `.redacted.txt` when regeneration would truncate.
- Protected-site DOCX `.redacted.docx` handoff is limited to safe regenerated output; malformed, encrypted, unsupported, no-text, macro, legacy, or truncated DOCX cases do not hand off `.redacted.docx` or raw DOCX.
- Protected-site XLSX `.redacted.xlsx` handoff is limited to safe regenerated output; malformed, encrypted, unsupported, no-text, macro, legacy, or truncated XLSX cases do not hand off `.redacted.xlsx` or raw XLSX.
- Scanner visual image redaction exports PNG only.
- Protected-site OCR is settings-controlled, enabled by default for supported image uploads, and can be turned off.
- Protected-site image visual upload uses `.redacted.png` only when OCR boxes are eligible.
- There is no scanned-PDF OCR, non-English OCR, remote OCR/backend, remote model call, image format preservation, layout-preserving PDF/DOCX/XLSX redaction, legacy/macro DOC/XLS support, or macro-enabled Office support.
