# LeakGuard 3.0 Full-Feature Reliability Results

## Verdict

`NO-GO` for a 3.0 release declaration. The implementation and automated Chrome/Edge gates are green, but the strict matrix correctly remains incomplete for exact-package authenticated and managed-enterprise cells. Firefox is outside the active scope.

Runtime and release-tooling candidate: branch `codex/leakguard-3.0-reliability`, commit `f706101` plus this evidence-only record.

## Fixed root causes

- WhatsApp/protected-site OCR broker readiness: a timed-out hidden broker frame remained cached, so later prepare attempts reused the same rejected readiness promise. The broker now discards only the matching timed-out frame and retries preparation once. Persistent failure remains fail-closed with the existing raw-image block and the terminal reason remains distinguishable from OCR recognition failure.
- Gemini/Grok panel mismatch: a verified first send changes the provider URL from its landing route to a conversation route, and tab initialization treated that expected navigation as a new session. Placeholder state was cleared before the panel refresh. A five-second, single-consumption verified-send carry preserves state only for the known Gemini/Grok route shapes; subframes cannot mutate the top-frame route.
- Same-tab file state: redaction mutations are serialized per tab so concurrent files cannot overwrite placeholder mappings; different tabs remain concurrent.
- Active release scope: package generation now cleans stale artifacts, checksums contain only Chrome consumer/enterprise ZIPs, and runtime-size reporting covers only those two targets. Edge consumes the Chromium extension path; Firefox is non-gating.
